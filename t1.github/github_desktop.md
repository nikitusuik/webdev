
## Демонстрация выполнения команд Git в GitHub Desktop (Скриншоты с Виндоус, тяжело на маке скринить)

### 1. Клонирование репозитория (git clone)

Клонирование репозитория выполняется через меню:

File → Clone repository

Пользователь выбирает репозиторий на GitHub и указывает локальную папку для сохранения.

Скриншот:  


![Clone repository](screenshots/clone.png)

---

### 2. Отслеживание изменений (git status)

После внесения изменений в файлы GitHub Desktop автоматически отображает список изменённых файлов и их состояние.

Это соответствует выполнению команды:
git status


**Скриншот:**


![Changes view](screenshots/changes.png)

---

### 3. Добавление файлов в индекс (git add)

Все изменённые файлы автоматически попадают в staging area. Пользователь может выбрать файлы или отдельные строки для коммита.

Соответствует команде:

```bash
git add .
Скриншот:
Отмеченные файлы, подготовленные к коммиту.

![Staging area](screenshots/staging.png)

---

### 4. Создание коммита (git commit)

Для создания коммита вводится сообщение и нажимается кнопка Commit to main.

Соответствует команде:

git commit -m "commit message"
Скриншот:
Окно создания коммита с введённым сообщением.

![Commit](screenshots/commit.png)

---

### 5. Отправка изменений в удалённый репозиторий (git push)

После создания коммита изменения отправляются в GitHub с помощью кнопки Push origin.

Соответствует команде:

git push
Скриншот:
Кнопка Push origin или сообщение об успешной отправке.

![Push](screenshots/push.png)

---

### 6. Получение изменений из удалённого репозитория (git pull)

Для получения обновлений используется кнопка Fetch origin / Pull origin.

Соответствует команде:

git pull
Скриншот:
Процесс получения изменений или обновлённое состояние репозитория.

![Pull](screenshots/pull.png)

---

### 7. Работа с ветками (git branch, git checkout, git merge)

В GitHub Desktop можно:

* создать новую ветку
* переключаться между ветками
* выполнять слияние веток через интерфейс

Соответствующие команды Git:

git branch
git checkout branch_name
git merge branch_name
Скриншот:
Меню выбора веток и процесс слияния.

![Branches](screenshots/branches.png)

## Особенности GitHub Desktop

### Преимущества

* удобный графический интерфейс
* наглядный просмотр изменений
* не требует использования терминала
* подходит для начинающих пользователей

### Ограничения

* ограниченные возможности для сложных операций Git
* не предназначен для автоматизации
