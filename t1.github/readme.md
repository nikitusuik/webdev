# Работа с Git и GitHub — повторение шагов за преподавателем

В данной работе я повторял действия преподавателя по шагам, осваивая базовые команды Git и работу с удалённым репозиторием GitHub.

---

## 1. Инициализация репозитория и первый push

На начальном этапе я попытался отправить изменения в удалённый репозиторий:

git push -u origin master
````

### Возникшие проблемы

* Ошибка:

  ```text
  src refspec master does not match any
  * В репозитории не было ни одного коммита
* Ветка master не существовала (используется main)
* GitHub не принимал пароль при выполнении git push

### Что было сделано

* Добавил файлы в индекс:

  
  git add .
  * Создал первый коммит:

  
  git commit -m "initial commit"
  * Выполнил push в ветку main:

  
  git push -u origin main
  * Для аутентификации использовал Personal Access Token

### Результат

* Первый коммит успешно загружен в GitHub
* Локальная ветка main связана с origin/main

---

## 2. Работа с ветками

Была выполнена работа в отдельной ветке.

### Действия

* Просмотр веток:

  
  git branch
  * Переход в ветку clean_up:

  
  git checkout clean_up
  
### Изменения

* Удалены файлы:

  
  git rm readme-after.txt
  git rm readme-now.txt
  * Создан коммит:

  
  git commit -m "DELETED"
  
### Результат

* Изменения зафиксированы в ветке clean_up

---

## 3. Слияние веток и удаление временной ветки

### Действия

* Возврат в основную ветку:

  
  git checkout main
  * Слияние ветки clean_up:

  
  git merge clean_up
  * Удаление временной ветки:

  
  git branch -d clean_up
  * Отправка изменений в GitHub:

  
  git push
  
### Результат

* Слияние прошло в режиме fast-forward
* Ветка main обновлена на GitHub

---

## 4. Получение изменений из удалённого репозитория

### Действие

git pull origin main
### Результат

* Локальный репозиторий синхронизирован с GitHub
* Изменения успешно применены
